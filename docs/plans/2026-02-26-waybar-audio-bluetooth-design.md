# Waybar Audio & Bluetooth Module Revamp

## Problem
The built-in `pulseaudio` and `bluetooth` waybar modules are functional but generic.
Volume shows only icons (no percentage), bluetooth click does nothing, and there's no
per-stream audio control or quick bluetooth connect.

## Design

### Volume → `custom/audio`

**Status script**: `audio-status` — polls every 2s + signal 14 for instant user-action updates.

**Display** (text field):
- Normal: `icon vol` — e.g. ` 72` (no % symbol, implied)
- Muted: ` ──` in error color
- Source active: append mic icon ` `; source muted: ` ` in error color
- Icons: ` ` (0), ` ` (1-50), ` ` (51-100), ` ` (>100)

**Tooltip**: Default output device name + active audio streams list (app, vol%, output).

**Classes**: `muted`, `source-muted`, `high` (vol > 100%).

**Interactions**:
- Scroll up/down: ±5% volume (keep `audio-volume` script)
- Click: `audio-switcher` TUI (preserved as-is)
- Right-click: `audio-mixer` TUI (NEW — per-stream control)
- Middle-click: toggle mute

### Bluetooth → `custom/bluetooth`

**Status script**: `bluetooth-status` — polls every 5s + signal 15.

**Display**:
- Controller off: `󰂲` (dim)
- On, no connections: `󰂯` (default color)
- Connected, no battery: `󰂱` (accent color)
- Connected with battery: `󰂱 85` (accent + level, color shifts at low battery)
- Multiple devices: `󰂱 2` (count)

**Tooltip**: Device name(s), battery %, codec/profile, controller info.

**Classes**: `off`, `on`, `connected`, `low-battery`.

**Interactions**:
- Click: `bluetooth-picker` TUI (NEW — quick connect/disconnect)
- Right-click: `bluetoothctl` interactive in foot

### Audio Mixer TUI (`audio-mixer`)

fzf in foot terminal. Lists all active PulseAudio sink-inputs with:
- App name (from `application.name` property)
- Unicode volume bar: `━━━━━━━━░░ 82%`
- Current output device: `→ Speakers`

fzf keybinds for real-time control:
- `right`/`left`: volume ±5% (execute-silent + reload)
- `enter`: open output device sub-picker for selected stream
- `m`: toggle mute (execute-silent + reload)
- `esc`/`q`: quit

### Bluetooth Picker TUI (`bluetooth-picker`)

fzf in foot terminal. Lists paired devices:
- `● Device Name  Connected  85%` or `○ Device Name  Disconnected`
- Select to toggle connect/disconnect
- Header option to scan for new devices

## Files

### New
- `.local/bin/audio-status` — waybar volume status
- `.local/bin/audio-mixer` — per-stream TUI
- `.local/bin/bluetooth-status` — waybar bluetooth status
- `.local/bin/bluetooth-picker` — quick-connect TUI

### Modified
- `.config/waybar/config` — replace built-in modules with custom
- `.local/share/theme-engine/templates/waybar-style.css.tmpl` — new styles
- `.config/waybar/style.css` — new styles (generated by theme engine)
- `.config/hypr/hyprland.conf` — add signals to media key binds

### Preserved
- `.local/bin/audio-switcher` — untouched
- `.local/bin/audio-volume` — untouched (or minor signal addition)
